# Presentation: Building a Community Space with Discord
> Amnesty International Thailand | ≤45 min | 10 slides | Philosophy-first

---

## Key Decisions (from planning)
- **Audience:** Amnesty Thailand — activists/volunteers, ไม่ใช่ developer
- **Thesis:** Open community space = พื้นที่แห่งความไว้วางใจ (ไม่ใช่แค่ visibility แต่คือ trust + transparency)
- **Tone:** Philosophy-heavy, technical detail มีใน คู่มือแยก อย่าซ้ำ
- **Demo:** เปิด PPLE volunteers server สด (slide 6)
- **Admin vs Moderator:** Admin = technical (1-2 คน), Moderator = community care (หลายคน)

> *"สิ่งที่องค์กรอื่นต้องจ่ายเงิน — Discord ให้ฟรี และมักดีกว่า"*

### Slide 2 — ปัญหาของ NGO: ทำงานในมืด
**เวลา:** 3 นาที

- คนอยากช่วย แต่ไม่รู้ว่ากำลังทำอะไรอยู่
- งานสำคัญเกิดในกลุ่มเล็กๆ ไม่มีใครเห็น
- สมาชิกใหม่เข้ามาแล้วเงียบ ไม่รู้จะเริ่มตรงไหน
- ผู้นำ burnout เพราะแบกงานไว้คนเดียว

> *"ปัญหาไม่ใช่ขาดคน — ปัญหาคือคนเห็นกันไม่ถึง"*

### Slide 3 — Philosophy: พื้นที่แห่งความไว้วางใจ
**เวลา:** 5 นาที  
*(slide หลัก — ใช้เวลาที่นี่)*

การเปิดพื้นที่ไม่ใช่แค่ "ให้คนเห็น" แต่คือ **การแสดงออกถึงความไว้วางใจ**

- เปิดเผย = เราไว้วางใจสมาชิก
- โปร่งใส = สมาชิกไว้วางใจองค์กร
- เมื่อคนเห็นว่าเราทำงานจริง → เขาจะยื่นมือมาช่วย
- เมื่อคนเห็นบรรยากาศการทำงาน → เขาอยากมีส่วนร่วม

> *"Community ไม่ได้สร้างจากการชวน แต่สร้างจากการให้คนเห็น"*

สำหรับ Amnesty — องค์กรที่ทำงานเรียกร้องความโปร่งใสจากผู้อื่น  
การทำงานแบบ open ภายในคือการ **ปฏิบัติตามที่เชื่อ**

---

### Slide 4 — หลักออกแบบ: Public First
**เวลา:** 5 นาที

**กฎหลัก:** ห้อง private คือข้อยกเว้น ไม่ใช่ default

- ทำงานในห้องที่คนเห็นได้ → กระทู้สื่อ, แผนงาน, meeting notes
- ห้อง private ไว้สำหรับ: ข้อมูลส่วนบุคคล, กรณีอ่อนไหว เท่านั้น
- ถ้าไม่แน่ใจ → เลือก public เสมอ

**โครงสร้างพื้นที่แนะนำ:**
```
📢 PUBLIC — ทุกคนเห็น
  #ประกาศ
  #ข่าวสาร
  #กระทู้สาธารณะ
  #กิจกรรม

🔒 MEMBER — สมาชิกเท่านั้น
  #ทีมงาน
  #แผนงาน
  #meeting-notes  ← ทำงานตรงนี้ ให้คนเห็นกระบวนการ

🛡️ PRIVATE — จำกัดมาก
  #admin
  #กรณีอ่อนไหว
```

### Slide 5 — เริ่มสร้าง: โครงสร้าง Server
**เวลา:** 3 นาที

**3 สิ่งแรกที่ต้องมี:**
1. **Channel หลัก** — #ประกาศ + #ห้องกลาง (public)
2. **Role พื้นฐาน** — สมาชิกทั่วไป / ทีมงาน / admin
3. **Welcome flow** — คนเข้ามาต้องรู้ว่าต้องทำอะไรก่อน

**อย่าสร้างห้องมากเกินไปตั้งแต่แรก** — เริ่มน้อย ขยายตามการใช้จริง  
Server ที่มีห้องว่าง 20 ห้อง ทำให้ดูเงียบและน่ากลัว

### Slide 7 — Admin & Moderator: คนดูแลพื้นที่
**เวลา:** 5 นาที

**แยกบทบาทออกจากกัน:**

| | Admin | Moderator |
|---|---|---|
| จำนวน | 1-2 คน | หลายคน |
| หน้าที่หลัก | ตั้งค่า server, bot, permissions | ดูแลบรรยากาศ, ต้อนรับ, กระตุ้นบทสนทนา |
| ทักษะ | Technical | Community sense |
| ความถี่ | Setup ครั้งแรก + แก้เป็นครั้งคราว | ทุกวัน |

> *"Admin ที่ดีต้องมีทั้ง 2 ทักษะ — แต่ในความจริงหาคนแบบนั้นยาก  
> ถ้าต้องเลือก — community sense สำคัญกว่า เพราะ technical ฝึกได้"*

Moderator ดี = คนที่รู้ว่าเมื่อไหรควรเข้าไป และเมื่อไหรควรถอย

### Slide 8 — Engagement Tools: เครื่องมือสร้างการมีส่วนร่วม
**เวลา:** 4 นาที

เครื่องมือที่ Discord มีให้และ NGO ใช้ได้ทันที (ไม่ต้อง technical):

- **@mention** — เรียกคนถูกคน ถูกเวลา ไม่ spam ทั้งกลุ่ม
- **Forum channels** — กระทู้ที่คนตอบกลับได้ เหมาะกับ discussion
- **Events** — สร้าง event, คน RSVP, แจ้งเตือนอัตโนมัติ
- **Threads** — คุยลึกโดยไม่รบกวนห้องหลัก
- **Stage channels** — town hall, พูดคุยสาธารณะ
- **Notification settings** — คนปรับได้เอง ไม่ถูก spam

### Slide 9 — Advanced Layer: AI, Bot, และ CRM
**เวลา:** 4 นาที

*(ไม่เน้น technical — เน้นว่ามันทำอะไรได้)*

**Bot + Automation:**
- สมาชิกติดยศ role ได้เอง (self-assign) ตามความสนใจ
- แจ้งเตือนอัตโนมัติเมื่อมีกิจกรรมใหม่
- ดึงข้อมูลจากระบบหลังบ้าน (CRM, ระบบสมาชิก, บริจาค)

**AI ใน Discord:**
- ตอบคำถามพื้นฐาน ลด load ทีมงาน
- สรุป discussion ยาวๆ
- ช่วย onboard สมาชิกใหม่

**Sync กับระบบ:**
- เชื่อม Discord ID กับข้อมูลสมาชิกจริง
- รู้ว่าใครบริจาค, ใครอาสา, ใครขาดการติดต่อ

> *"เมื่อ Discord เชื่อมกับระบบหลังบ้าน — มันไม่ใช่แค่แชท มันคือ hub ขององค์กร"*

### Slide 10 — ทำยังไงให้ Community โตเอง
**เวลา:** 4 นาที + Q&A

**Community โตเองได้เมื่อ:**

1. **คนเห็นว่ามีคนอื่นทำงานอยู่** → อยากเข้ามาร่วม
2. **เข้าร่วมได้ง่าย** → ไม่มีขั้นตอนซับซ้อน
3. **ได้รับการตอบสนอง** → พูดแล้วมีคนฟัง ไม่เงียบ
4. **มี quick win** → ทำอะไรเล็กๆ แล้วเห็นผลได้เลย

**สิ่งที่ Moderator ต้องทำในช่วงแรก:**
- ทำงานในที่สาธารณะอย่างสม่ำเสมอ (seed activity)
- ต้อนรับสมาชิกใหม่ทุกคนด้วยชื่อจริง
- ตั้งคำถามในห้องเพื่อกระตุ้นให้คนตอบ
- ยกย่องคนที่มีส่วนร่วม publicly

**เริ่มได้เลยวันนี้:**
1. สร้าง server (5 นาที)
2. ตั้ง 3 ห้องแรก + 2 role
3. ชวนคนใน team 5 คนแรก → ทดลองทำงานใน public
